# New Project Wizard verification matrix

Automated coverage lives in:

- `ArmeriaWizardTemplateCoverageTest` — every wizard library id has a matching `hasLibrary(...)` block in all three build templates.
- `ArmeriaWizardTemplateRenderingTest` — renders Velocity templates for the representative wizard combinations below.

## Representative scenarios

| # | Build | Language | Libraries | Expected in generated build file |
|---|-------|----------|-----------|----------------------------------|
| 1 | Gradle (Kotlin DSL) | Kotlin | `armeria-grpc`, JUnit 5 | `kotlin("jvm")`, `armeria-grpc`, `armeria-junit5`, `useJUnitJupiter()` |
| 2 | Gradle (Groovy) | Scala | `armeria-scala_2.13` | `id 'scala'`, `armeria-scala_2.13`, `scala-library:2.13.8` |
| 3 | Maven | Kotlin | `armeria-tomcat8` | `<artifactId>armeria-tomcat8</artifactId>`, `kotlin-maven-plugin` |
| 4 | Gradle (Kotlin DSL) | Kotlin | `armeria-spring-boot3-starter` | `armeria-spring-boot3-starter` |

## Files always generated by `ArmeriaModuleBuilder.getAssets`

| Build type | Files |
|------------|-------|
| Gradle (Kotlin DSL) | `build.gradle.kts`, `settings.gradle.kts`, `gradle.properties` |
| Gradle (Groovy) | `build.gradle`, `settings.gradle`, `gradle.properties` |
| Maven | `pom.xml` |
| All (Java/Kotlin) | `src/main/{java\|kotlin}/<package>/Main.{java\|kt}`, `src/main/resources/logback.xml`, `src/test/{java\|kotlin}/<package>/MainTest.{java\|kt}` when JUnit 5 is selected, empty source directories |
| Scala | `src/main/resources/logback.xml` and empty `src/main/scala/<package>/` / `src/test/scala/<package>/` directories only (no sample Main or test) |

Not generated by this plugin: Gradle Wrapper, Java toolchain pins.

## Optional manual QA (`runIde`)

1. `./gradlew :plugin:runIde`
2. **New Project** → Armeria → run each row in the table above.
3. Confirm the generated tree matches the table and `./gradlew build` (or `mvn -q test-compile`) succeeds.
